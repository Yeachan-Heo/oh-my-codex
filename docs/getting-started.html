Getting Started

End-to-end setup for first-time users.

Prerequisites

  • Node.js 20+ and npm.
  • OpenAI Codex CLI installed and authenticated.
  • Git (recommended) for project workflows.

Installation

npm install -g oh-my-codex
omx setup
omx doctor

Windows Quick Start

If global npm install/link is restricted by policy, run OMX directly from the repo:

node .\bin\omx.js setup
node .\bin\omx.js doctor
node .\bin\omx.js team 2:executor "test task"

On Windows, team mode automatically uses process workers when tmux is unavailable.

First Run Walkthrough

  1. Create or enter a project directory.
  2. Run omx setup to install prompts, skills, AGENTS.md, and configuration.
  3. Start Codex CLI with codex.
  4. Try an agent prompt: /prompts:architect "analyze auth module".
  5. Try a workflow skill: $team "fix all TypeScript errors".

Troubleshooting

IssueWhat to check
omx command not foundRe-run global install and ensure npm global bin is in PATH.
npm EPERM/EACCES during global installUse a local npm cache, run elevated shell, or run via node .\bin\omx.js ... from the repository.
Prompts not availableConfirm files exist in ~/.codex/prompts/ and re-run omx setup.
Skills not loadingVerify ~/.agents/skills/*/SKILL.md files were installed.
Doctor reports config issuesRun omx doctor and apply the suggested fixes.
worker_process_missing in omx doctor --teamA worker PID is stale. Run omx team shutdown <team-name> and start the team again.
